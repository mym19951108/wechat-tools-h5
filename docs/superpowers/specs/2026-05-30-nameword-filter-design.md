# Poetry Name — Semantic Quality Filter

**Date**: 2026-05-30  
**Status**: draft  

## Problem

Jieba `coherent` flag only ensures a word is dictionary-recognized, not that it has poetic value as a name. Examples of valid jieba words that make poor names:

| Poem | Extracted | Issue |
|---|---|---|
| 金山+西见烟尘飞 | 山西 | Jieba-valid, but it's a province name, not poetic |
| 只在此山中，云深不知处 | 山中 | "X+中" pattern — too mundane |
| 蜀主窥吴幸三峡，崩年亦在永安宫 | 永安 | Good name, but source poem is about death |
| 狐狸何足道，豺虎正纵横 | 而已 | Function word combo |

## Design

### Five-Filter System

**File**: `scripts/build-poetry-db.js`

Add `isNameworthy(chars)` check, called in `isValid()`. Rejected combos never enter poetry-map.

#### Filter 1: Pure Place Names (automatic)
Blocklist of Chinese administrative division names:
```
山西 山东 河南 河北 湖南 湖北 广东 广西 江西 江苏 浙江 四川 贵州 云南 福建 安徽 陕西 甘肃 宁夏 西藏 新疆 内蒙古 海南 台湾 北京 上海 天津 重庆 江南 岭南 中原 关东 关西
```

#### Filter 2: Directional Suffixes (automatic)
Pattern: `[any char] + 中/上/下/里/外/前/后/东/南/西/北` → reject
```
山中 江上 月下 林里 门外 云前 雨后 ... (generated by regex)
```

#### Filter 3: Function Words (automatic)
Blocklist of common Chinese function word pairings:
```
而已 之于 然而 所以 可以 于是 至于 及其 以及 以此 因此 何必 何不 无非 不论 不管
```

#### Filter 4: Pronoun/Interrogative (automatic)
Pattern: starts with or contains pronoun/question word + another char:
```
其中 何处 此人 此时 何年 那时
```
Implemented as blocklist:
```
其中 何处 此人 此时 何年 何时 此日 其地
```

#### Filter 5: Historical Proper Names (manual curated blacklist)
Specific historical names that are either bad connotations or too specific:
```
永安 长安 建安 永和 太和 开元 天宝 贞观 洪武 永乐 乾隆 康熙 昭和 平成
```

### Integration

```js
function isNameworthy(chars) {
  if (NAME_BLOCKLIST.has(chars)) return false                   // Filters 1, 3, 4, 5
  const c1 = chars[0], c2 = chars[1]
  if ('中上下里外前后东西南北'.includes(c2)) return false       // Filter 2
  if ('中上下里外前后东西南北'.includes(c1)) return false       // Filter 2 (reverse)
  return true
}
```

Called in `isValid()`:
```js
if (!isNameworthy(c.chars)) continue  // Added before entry push
```

## Affected Files

| File | Change |
|---|---|
| `scripts/build-poetry-db.js` | Add `isNameworthy()` with 5 filters; call in `isValid()` → `processPoems()` |

## Implementation Steps

1. Edit `scripts/build-poetry-db.js` — add filters
2. Run `node scripts/build-poetry-db.js` — regenerate poetry-map
3. Run `npm test` — verify all pass
