# 诗词名字库 v3：chinese-poetry 预处理 + 五行词组索引

## 概述

用 chinese-poetry 开源库替换手工维护的 poetry-map.js。编写一次性预处理脚本，扫描诗经/楚辞/唐诗三百首/宋词三百首，提取所有可做名字的 2 字词组，按五行分组存储。同时将 names.js 字池从 45 字扩容到 150-200 字。

## 架构

```
scripts/build-poetry-db.js (一次性脚本)
  ├── git clone chinese-poetry (浅克隆 json 目录)
  ├── 扫描: 诗经(300首) + 楚辞(65首) + 唐诗三百首(~320首) + 宋词三百首(~300首)
  ├── 提取 3 种模式 2 字组合
  ├── 白名单过滤 + 禁用字过滤
  ├── 部首法 + 手动表 标五行
  ├── 去重 → 按五行分组
  │
  ├── 输出 → src/data/poetry-map.js (25 组五行组合)
  └── 输出 → src/data/names.js (扩展版, 150-200 字)
```

## 提取规则

**三种模式**（以 "落霞与孤鹜齐飞" 为例）：

| 模式 | 示例 | 产出 |
|------|------|------|
| 相邻 | 落霞/霞与/与孤/孤鹜/鹜齐/齐飞 | 落霞 |
| 隔一字 | 落与/霞孤/与鹜/孤齐/鹜飞 | 落与 |
| 首尾 | 落飞 | 落飞 |

**三层过滤**：
1. 白名单过滤：两字都在取名白名单中 → 通过
2. 禁用字过滤：两字都不在禁用字表中 → 通过
3. 组合过滤：虽单字合法但组合负义（如死+亡→死亡） → 排除

## 白名单与禁用字

**白名单来源**：
- 原 names.js 45 字 (保留)
- 现代新生儿高频取名用字 (~80 字)
- 康熙字典寓意好的字 (~60 字)
- 部首明确的字优先
- 总计 150-200 字

**五行标注规则**：
1. 部首法：金钅→金, 木→木, 氵水→水, 火灬→火, 土→土
2. 康熙字典五行参考
3. 无法标注 → 标记"未知"，不进入诗词库

**禁用字表**：
```
悲恨死伤残亡绝弃败衰落病痛苦怨恨仇辱耻废毁灭丧败
凶邪恶毒灾祸咎耻……
```

## BaziNameEngine 运行时改造

```
八字分析 → 喜用神=['水', '木']
  │
  ├── 计算优先级五行组合:
  │   '水水'(同主喜用): 优先
  │   '木木'(同喜用): 优先
  │   '水木'(跨喜用): 次优
  │   '木水': 次优
  │   '水金'(喜用+中性): 可选
  │   ...
  │
  ├── 从 wuxingPoetryNames 按组合取词组
  ├── 不足时 fallback 到 仅五行匹配(旧逻辑)
  │
  └── 打分: 喜忌(40) + 同句诗(30) + 音韵(15) + 寓意(15)
```

## 评分体系保持不变

| 维度 | 分 | 规则 |
|------|-----|------|
| 喜忌 | 40 | 两字同主喜用40, 两字喜用35, 一字喜用25, 中性15 |
| 诗词 | 30 | 同句30, 同诗25, 仅有出处20 |
| 音韵 | 15 | 笔画交替+差3-12得15, 仅交替12, 仅差合适10, 其他7 |
| 寓意 | 15 | 两字≥95得15, ≥90得13, ≥85得11, 其他9 |
| **总分** | **100** | 85+bonus 保底模式 |
